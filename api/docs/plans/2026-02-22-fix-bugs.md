### Task 4: Fix deprecated Pydantic v1 Config class

`src/config/settings.py` uses the old Pydantic v1 inner `class Config` pattern. Pydantic v2 uses `model_config = SettingsConfigDict(...)`. This generates a deprecation warning on every startup.

**Files:**
- Modify: `src/config/settings.py:28-30`

**Step 1: No new test needed** — the existing startup warning already proves the issue. After fix, run the full suite to confirm no regressions.

**Step 2: Fix settings.py**

Change `src/config/settings.py` from:

```python
from pydantic_settings import BaseSettings


class Settings(BaseSettings):
    # ...fields...

    class Config:
        env_file = ".env"
        env_file_encoding = "utf-8"
```

To:

```python
from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8")

    # ...fields (unchanged)...
```

The full updated file:

```python
from functools import lru_cache

from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    model_config = SettingsConfigDict(env_file=".env", env_file_encoding="utf-8")

    # Server
    environment: str = "development"
    port: int = 8000
    host: str = "0.0.0.0"

    # Database
    database_path: str = "data/tracker.db"

    # Uploads
    uploads_path: str = "data/uploads"

    # Logging
    log_level: str = "info"

    # Features
    enable_docs: bool = True

    # API
    api_title: str = "Personal Tracker API"
    api_version: str = "1.0.0"


@lru_cache
def get_settings() -> Settings:
    return Settings()
```
